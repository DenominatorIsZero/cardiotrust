# AGENTS.md

CardioTrust is a Rust research application for cardiac electrophysiological simulation (Bevy + egui desktop, WASM, and batch planner).

## Commands

```bash
just run / just release   # Run native desktop app
just test                 # Run all tests (nextest)
just test-all             # Include #[ignore]d tests
just lint                 # clippy-tracing span check + cargo clippy
just fmt                  # cargo +nightly fmt (required before committing)
just check                # cargo check + clippy -D warnings
just ci                   # fmt-check + check + test

# Single test:
cargo nextest run <test_name>
cargo nextest run --ignored <test_name>
```

> Some tests require data generated by running the app first (`just run`).

## Non-Obvious Rules

**Tracing:** Every public function requires `#[tracing::instrument]` — enforced by `clippy-tracing` at lint time. Use `skip_all` for functions taking large structs (ndarray arrays, Model, etc.).

**Error handling:** No `unwrap()` ever. `expect()` is only acceptable in Bevy systems and thread closures that cannot return `Result`. Everywhere else return `anyhow::Result` and propagate with `?`.

**Imports:** `rustfmt.toml` sets `imports_granularity = "Crate"` and `group_imports = "StdExternalCrate"`. Run `just fmt` — do not hand-sort.

**Algorithm safety:** Do not change computational behaviour when refactoring. GPU and CPU implementations must stay in sync.

**Module layout:** `foo.rs` declares the module; `foo/` holds its submodules. File names mirror the primary type they define.

**`results/` and `logs/`** are gitignored and can be very large — never commit them.
